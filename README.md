# 📱 MagIC GitHub

## 🎯 Objectif
Reprendre et finaliser une application Android existante permettant de lister des utilisateurs GitHub, en corrigeant les tests et en implémentant les fonctionnalités manquantes.

---

## 🧠 Contexte
Ce projet s’inscrit dans un scénario professionnel simulé :

Intégration en tant que développeur Android junior chez O’Company, avec pour mission de reprendre une application incomplète développée par un ancien développeur.

---

## 🧪 Technologies utilisées
- Java
- Android SDK
- JUnit (tests unitaires)
- Espresso (tests instrumentalisés)
- Git / GitHub

---

## ⚙️ Fonctionnalités

### Fonctionnalités existantes
- Affichage d’une liste d’utilisateurs GitHub

### Fonctionnalités implémentées
- ✅ Suppression d’un utilisateur
- ✅ Ajout d’un utilisateur aléatoire

---

## 🧪 Tests

### Situation initiale
- Tests unitaires et instrumentalisés existants
- ❌ Échec des tests

### Travail réalisé
- Analyse des erreurs
- Correction du code source
- Mise à jour des comportements

### Résultat final
- ✅ Tous les tests passent avec succès

---

## 🧱 Architecture

Le projet repose sur :
- Un repository pour la gestion des données (`UserRepository`)
- Une séparation logique entre les couches
- Une gestion de liste d’utilisateurs

---

## 🔄 Gestion de version (Git)

- Création d’un repository GitHub
- Utilisation de branches :
  - `main` : version stable
  - `develop` : développement des fonctionnalités
- Implémentation des fonctionnalités dans `develop`
- Création et validation d’une Pull Request vers `main`
